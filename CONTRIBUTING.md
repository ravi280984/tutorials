Thank you for your interest in contributing to this repository.

Guidelines

- Fork the repository and create feature branches for changes.
- Keep PRs small and focused. Each PR should have a clear description and link to any related issue.
- Write tests for new functionality and ensure existing tests pass.
- Follow code style: run `black .` and `isort .` before opening a PR.
- Use meaningful commit messages and reference issues when appropriate.

Submitting Changes

1. Fork -> branch -> commit -> push.
2. Open a Pull Request against `main` with a clear title and description.
3. Address review comments; maintainers will merge once checks pass.

Security

If you discover a security issue, please open a private issue or contact the maintainers directly. Do not include secrets in PRs or issues.
